In the time since I first started looking into S3 (maybe 2 years ago?) it seems like not a lot of progress has been made on a really good library for manipulating S3 in PHP. I imagine this is partly due to the fact it is somewhat trivial to roll your own if you only focus on the features you actually need. Maybe if AWS would actually release and maintain a library themselves it would help.

Meanwhile, we're stuck wading through any number of library options, trying to determine which ones are less half baked than any of the others.

So far I've found two that look promising, php-aws and Amazon S3 PHP Class (amazon-s3-php-class on Google code). php-aws looks clean, though maybe not as many features. I was really excited about amazon-s3-php-class until I looked more closely at the code. Really not super stoked on the fact that the object looks to be a front to static data.

If neither of these implementations work out, I may have to write my own lite weight S3 library, possibly referencing large chunks of either of these two libraries.

Bundling an S3 implementation with Safsi might be required anyway, as it would limit the amount of dependencies on other third party libraries. I was sort of hoping to leverage someone else's code here, though. Otherwise I'll just end up contributing Yet Another S3 Library For PHP.

Here is to hoping that the CloudFiles implementation provided by Mosso live up to my requirements.
