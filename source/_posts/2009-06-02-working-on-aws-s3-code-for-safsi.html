I've decided to work on my own AWS S3 implementation for Safsi. No real idea how far out the AWS S3 related code is from being checked in, but it is definitely high priority on my list of personal projects as several projects could leverage this code now.

I have been borrowing a lot of ideas from php-aws and Amazon S3 PHP Class (amazon-s3-php-class on Google code). Anyone looking for general AWS libraries would do well to check out either of these libraries!

I have every intention of keeping the Safsi AWS S3 and Cloud Front code standalone in the sense that the AWS related code should be usable without actually using the Safsi file system interfaces. Will anyone use them if they are only available as a part of the Safsi distribution? I am not sure, but I do like the idea of keeping the AWS interface code uncoupled from the Safsi file system interfaces.
