A few days ago Alvaro had asked me if there were any docs for Skittle. At the time there was none. As of last night this has changed. Skittle is sporting fresh phpdoc and a cheat sheet. The phpdoc pushed my timetable forward for getting a standalone website for Skittle as I wanted the phpdoc for Skittle to live somewhere other than the Halo website.

So yeah, as of last night, skittle-php.org is alive.

On a somewhat related note, Ohloh's code analysis is pretty cool. If not a bit damning. Proud of my new projects, I happily enlisted the subversion repositories only to discover that it puts up nice little yield/warning signs on account of my present lack of commenting and documentation. I took it as a challenge. And I quote:

Across all PHP projects on Ohloh, 30% of all source code lines are comments. For skittle-php, this figure is 53%.

This very impressive number of comments puts skittle-php among the best 10% of all PHP projects on Ohloh.

A high number of comments might indicate that the code is well-documented and organized, and could be a sign of a helpful and disciplined development team.

Yeah. Take that, Ohloh.

Seriously, though. It seems like it might be nice to have some sort of external accountability on my projects. On that note, when am I going to set myself on the much larger tasks of documenting Repose (5% commented), Substrate (1% commented) and Halo (6% commented)?
