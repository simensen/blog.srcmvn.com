Less than a week ago, I started working on rebuilding Repose, my PHP ORM persistence layer, from the ground up.

The original Repose worked pretty well, but it was very much an experiment on whether or not doing so was possible. With this recent rebuild, I have been able to apply a lot of things from recent experiences with SQLAlchemy to what I had already known about Hibernate. The result? A fundamental change in the core principles behind Repose. Changes that are going to be much better in the long run.

The only major downside right now is that there was one major change made to the requirements of model objects. Repose is no longer going to support protected properties for PHP < 5.3. This is a huge bummer to me, but it is the only way to really make Repose shine.

Once versions of PHP that support ReflectionProperty::setAccessible() become more mainstream, people will be able to start using Repose on models with protected and private properties without a problem. And that will be awesome.
