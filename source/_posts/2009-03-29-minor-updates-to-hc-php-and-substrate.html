I checked in a couple of minor changes to hc-php and to Substrate tonight. Two implementations of hc_configuration_IConfiguration were added, hc_configuration_PdoConfiguration and hc_configuration_CompositeConfiguration. Both are pretty naieve examples of implementation hc_configuration_IConfiguration (both are backed by hc_configuration_MapConfiguration) but they get the job done. The following is an example of being able to set defaults in a properties file and allowing for overriding defaults in a database.

https://gist.github.com/288213

In this case, the stringConfiguration object will combine the properties from the strings.properties file and the values stored in the database. I am using this with a new Halo View Helper named halo_StringViewHelper. Look for more information on that soon, but here is a quick usage example from a Skittle template.

https://gist.github.com/288216

First, the properties configuration is checked for name.long, and if it is not located the PDO configuration is checked. If neither configuration contains the key name.long, then ${name.long} is entered into the template.
