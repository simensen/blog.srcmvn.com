To use a 3rd party library with Halo, the most straightforward solution is to add the library files to a sub directory under vendors/. For the single site example distributed with Halo, the vendors folder consists of hc-php, skittle and substrate. Every directory in vendors/ is added to the include path, so requiring skittle_Skittle.php is as easy as require_once('skittle_Skittle.php'), or if using the hc class loader, hc_core_ClassLoader::load('skittle_Skittle');. While this is all fine and good, sometimes it is useful to use local development versions of a library that may not actually be maintained in the current projects distributrion. For instance, I include a version of hc-php, skittle, substrate, halo and repose with a project, but I also actively develop on each of these packages.
