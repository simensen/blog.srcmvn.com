Repose
After spending some time checking out the PHP ORM tool Outlet, I decided to take a stab at refactoring some of the code to handle protected and private properties differently. In the process, it became clear to me that some of the changes I was looking to make would require some hefty overhauling of many parts of the system. Rather than diving head first into an existing project and trying to make massive changes, I opted to start from scratch. Repose was born. Repose is a name I have long liked for a persistence layer, but Halogen Creative collectively decided to run with Pitchfork the last time we fired up a DAO project. From a marketing perspective, Halo and Pitchfork seemed to play with each other nicely than Halo and Repose. There is not a whole lot of information available yet, but there is a Repose website, a Repose Google Code hosted project and a Repose General Discussion Google Group setup if anyone is interested in seeing what is going on.
